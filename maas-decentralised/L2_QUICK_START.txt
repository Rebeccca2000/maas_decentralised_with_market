================================================================================
  COMMAND LINE OPTIONS
================================================================================

Network Selection:
  --network {localhost|optimism-sepolia|base-sepolia|arbitrum-sepolia}
  Default: localhost

Simulation Parameters:
  --steps <N>              Number of simulation steps (default: 100)
  --commuters <N>          Number of commuter agents (default: 20)
  --providers <N>          Number of provider agents (default: 10)

Flags:
  --debug                  Quick test (5 commuters, 3 providers, 20 steps)
  --big-test               Medium test (15 commuters, 8 providers, 50 steps)
  --no-plots               Skip plot generation for faster execution

Custom RPC:
  --rpc-url <URL>          Custom RPC endpoint
  --chain-id <ID>          Custom chain ID

================================================================================
  HOW IT WORKS
================================================================================

1. You run the script with --network flag
2. Script automatically configures blockchain_config.json
3. Simulation connects to selected network
4. Results are generated on that blockchain
5. Plots and metrics are saved locally

================================================================================

Console Output:

  - Transaction summaries
  - Advanced metrics

Generated Files:

  - simulation_plots_TIMESTAMP/cost_market_analysis.png
  - blockchain_config.json (updated with network settings)

================================================================================
  TROUBLESHOOTING
================================================================================

RPC Connection Issues:
  - Verify RPC URL is correct
  - Check network connectivity
  - Use --rpc-url to specify custom endpoint

Transaction Failures:
  - Check blockchain network status
  - Verify sufficient gas limits
  - Ensure contracts are deployed

Slow Performance:
  - Use --no-plots to skip visualization
  - Reduce --steps or agent counts
  - Use --debug mode for quick tests
